**规范要点**：
- **分级标准**：🔴 Blocker（必须修）/ 🟡 Suggestion（建议修）/ 💭 Nit（可选），覆盖 6 大维度（风格/逻辑/安全/性能/可维护/可测试），并给出本项目专属红线（fetch/ReadableStream/0.0.0.0 禁用等）
- **完整流程**：提交前 12 项自查清单 → 主/次审查 + 领域专家自动分配（基于 CODEOWNERS）→ 周轮值机制 → 三级争议升级路径（评论讨论 → 第二意见 → ADR）
- **场景化策略**：Hotfix 4 小时通道 / 大型重构（>500行）强制 ADR+分阶段 / 安全变更双审 / 新人 PR 主动 learning 反馈
- **度量指标**：效率（首响应 SLA 24h、合并周期 P95）+ 质量（缺陷逃逸率 <10%、回滚率 <5%）+ 协作（基尼系数 <0.4、争议升级率 <5%），明确"红线触发流程回顾"的硬指标
